## Executive Summary

Based on market analysis and v1.3 completion, v1.4 focuses on **offline-first experience** and **multi-user team collaboration**. These two features address key market gaps (隱私狂熱者 + 團隊領導者 segments) and enable future B2B growth.

#### Core Components

##### 2.1 身份驗證 (Authentication)
```
需求:
- 用戶註冊/登入 (email + password OR OAuth via GitHub)

- `users` table (id, email, password_hash, username, created_at, updated_at)
```

```
分享模式:
1. Private (default): 只有自己看得到
```

```
DB Schema:
- `comments` table (id, post_id, user_id, content, created_at)
- `reactions` table (id, post_id, user_id, emoji, created_at)
- `team_members` table (id, team_id, user_id, role, joined_at)
- `notifications` table (id, user_id, type, related_post_id, read_at)
```

## 2. Technical Architecture Changes

### Database Migration (SQLite → PostgreSQL)
```
Phase 1: Parallel Run (v1.3 final)
- SQLite remains as development database
- Add PostgreSQL support in config
- Run tests against both

Phase 2: Primary Switch (v1.4)
- PostgreSQL becomes default in production config
- SQLite still supported for local dev
- Migration script for existing SQLite users

Alembic Migrations:
1. Create users, sessions tables
2. Add user_id to posts
3. Create comments, reactions tables
4. Add team_members, notifications tables
5. Index optimization (user_id, created_at, shared_with)
```

### Backend API Changes
```
新增端點:

POST /api/v1/auth/register
POST /api/v1/auth/login
POST /api/v1/auth/logout
POST /api/v1/auth/refresh-token
GET  /api/v1/auth/profile
PUT  /api/v1/auth/profile

POST /api/v1/posts/{id}/share
GET  /api/v1/posts?shared_with_me=true
POST /api/v1/posts/{id}/comments
GET  /api/v1/posts/{id}/comments
POST /api/v1/posts/{id}/reactions
DELETE /api/v1/posts/{id}/reactions/{emoji}

POST /api/v1/export?format=json|csv|markdown|pdf

修改端點:

GET  /api/v1/posts → add user_id, privacy filter
POST /api/v1/posts → add privacy parameter
DELETE /api/v1/posts/{id} → validate ownership

Middleware Changes:
- 新增 JWTAuthMiddleware (check Authorization header)
- 新增 OwnershipMiddleware (verify user owns resource)
- 新增 RateLimitMiddleware (prevent export spam)
```

### Frontend Architecture Changes
```
新增 Context:
- AuthContext (manage login state, JWT token)
- TeamContext (manage team members, shared posts)
- NotificationContext (show alerts for activity)

新增 Routes:
- /login → LoginPage.jsx
- /register → RegisterPage.jsx
- /settings → SettingsPage.jsx
- /profile → ProfilePage.jsx
- /team → TeamManagementPage.jsx

新增 Components:
- PrivacySelector.jsx (dropdown: Private/Team/Public)
- CommentSection.jsx (nested comments)
- ReactionBar.jsx (emoji reactions)
- NotificationCenter.jsx (activity feed)
- TeamMembersModal.jsx (manage team)
```

## Appendix: User Stories for v1.4

### US-401: Team Member Collaboration
> As a team lead, I want to share my AI journal with team members so that we can collectively learn from the AI's perspective.

**Acceptance Criteria**:
- [ ] Can create Team and invite members via email
- [ ] Members see shared journal entries
- [ ] Can toggle privacy for each entry (Private/Team/Public)
- [ ] Receives notification when team member comments

### US-402: Complete Offline Experience
> As a traveler without internet, I want my journal app to work completely offline so I don't lose my thoughts.

**Acceptance Criteria**:
- [ ] Create/edit/delete posts offline
- [ ] View all past entries (cached)
- [ ] Sync automatically when back online
- [ ] No data loss on browser crash

### US-403: Data Export for Analysis
> As a researcher, I want to export 5 years of journal data in multiple formats so I can analyze AI behavior patterns.

**Acceptance Criteria**:
- [ ] Export to JSON, CSV, Markdown, PDF
- [ ] Filter by date range and mood
- [ ] Include AI decision metadata
- [ ] Download within 30s for 10k+ entries
